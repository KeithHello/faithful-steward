# 产品需求文档（PRD）— 忠心好管家 MVP

---

## 1. 项目资讯

| 栏位 | 内容 |
|------|------|
| **语言** | 繁体中文 |
| **技术栈** | iOS native（Swift/SwiftUI），语音辨识使用 iOS 内建 SFSpeechRecognizer |
| **专案名称** | `tithe_budget` |
| **原始需求** | 基于「什一奉献」理念的个人预算分配 iOS MVP app，支援语音/文字记账并自动匹配 7 大分类，提供多周期预算比例对比 |

---

## 2. 产品定义

### 2.1 产品目标

1. **降低记账门槛**：透过语音 + 文字快速输入，让使用者 5 秒内完成一笔记账，无需手动输入金额与分类。
2. **预算纪律可视化**：将「什一奉献」7 大类别的预算比例与实际花费比例做直观对比，帮助使用者快速发现超支/结余类别。
3. **轻量 MVP 验证**：专注核心记账 + 比例对比，快速上线收集使用者回馈，为后续模组化扩展奠定基础。

### 2.2 目标用户画像

| 维度 | 描述 |
|------|------|
| **身分** | 已建立「什一奉献」预算习惯的基督徒上班族 |
| **收入** | 月薪 25,000~35,000 台币（税后） |
| **痛点** | 现有记账 app 过于复杂、无法按固定比例对照实际花费；想知道自己是否「守住了预算比例」 |
| **使用情境** | 日常消费后随手记、月底/季末回顾预算执行状况 |
| **技术水平** | 一般手机使用者，不排斥语音输入 |

### 2.3 核心用户故事

- **US1**：作为使用者，我希望在消费后可以用**语音或文字**快速记录金额，app 自动辨识数字，然后我**一键选择分类**，这样我就不用花太多时间记账。
- **US2**：作为使用者，我希望在**月底**能看到每个分类的**实际花费比例 vs 预设预算比例**，一眼看出哪些类别超出、哪些低于预算，这样我可以调整下个月的消费习惯。
- **US3**：作为使用者，我希望可以切换不同时间周期（本月/近 3 个月/近半年/近一年），观察各分类比例的**趋势变化**，了解自己是否长期守住了预算纪律。
- **US4**：作为使用者，我希望在初次使用时能**设定本月预算总额**（预设可参考收入范围），让 app 自动计算出 7 个分类的预算上限。

## 3. 技术规范

### 3.1 功能需求池

| 优先度 | 功能 | 说明 |
|--------|------|------|
| **P0** | 快速记账（文字输入） | 输入金额/备注，自动识别数字，选择 7 大分类之一，记录日期与时间 |
| **P0** | 语音记账 | 语音转文字 → 提取金额 → 选择分类，等同文字记账流程 |
| **P0** | 7 大分类预设 | 内建 7 个不可删除的预设分类，比例固化（可于设定页微调） |
| **P0** | 预算比例对比 | 选择一个时间周期，显示各分类「实际比例 vs 预算比例」的条形图，标注超出/低于 |
| **P0** | 月预算总额设定 | 使用者设定当月可用预算总额，app 自动计算各分类上限 |
| **P1** | 多周期切换 | 本月 / 近 3 个月 / 近半年 / 近一年，四个周期切换检视 |
| **P1** | 分类比例客制化 | 允许使用者在设定中微调 7 大类的预设比例（总和须为 100%） |
| **P1** | 记账列表与历史 | 按日期排列的记账清单，支援依分类筛选与简单编辑/删除 |
| **P2** | 简易图表汇出 | 将比例对比图汇出为图片，方便分享 |
| **P2** | 分类颜色自订 | 允许使用者自订 7 大类的代表颜色 |
| **P2** | 记账备注与标签 | 每笔记账可加备注文字与自订标签 |

**交互关键点**：
- 记账页：语音辨识结果直接填入金额栏位，使用者只需选分类 → 点确认

- 周期切换用下拉选单，资料即时更新

### 3.3 完整 Use Case 表格

**UC1：文字记账**

| 流程 | 步骤 |
|------|------|
| **主流程** | 1. 使用者在 Tab1 文字栏输入金额（如「250」）<br>2. 系统自动辨识数字为 NT$250<br>3. 使用者点选分类按钮「食行」<br>4. 系统跳出确认弹窗「NT$250 → 食行」<br>5. 使用者点「确认」<br>6. 系统储存纪录，显示「记账成功」提示，清空输入栏 |
| **替代流程 A** | 3a. 使用者先选分类再输入金额 → 流程同上，不区分顺序 |
| **替代流程 B** | 4a. 使用者点「取消」→ 回到步骤 2/3，保留已输入资料 |
| **例外流程 E1** | 2a. 输入无法辨识的文字（如「吃晚餐」）→ 提示「请输入金额数字」 |
| **例外流程 E2** | 5a. 使用者未选分类就点确认 → 按钮反灰不可点击，提示「请选择分类」 |
| **例外流程 E3** | 6a. 金额为 0 或负数 → 提示「请输入有效金额」 |

**UC2：语音记账**

| 流程 | 步骤 |
|------|------|
| **主流程** | 1. 使用者长按麦克风按钮<br>2. 系统开始收音，显示声波动画<br>3. 使用者说「食行两百五」<br>4. 使用者放开按钮，系统进行语音辨识<br>5. 系统显示辨识结果「食行 250」，分类自动选中「食行」，金额填 NT$250<br>6. 使用者点「确认」<br>7. 系统储存纪录 |
| **替代流程 A** | 3a. 语音只包含金额（如「三百块」）→ 系统只填入金额，分类栏空白，等待手动选择 |
| **替代流程 B** | 5a. 语音辨识多关键字匹配（如「十一两百」）→ 系统填入分类 = 十一、金额 = 200 |
| **例外流程 E1** | 语音辨识失败（无声音/杂讯过大）→ 提示「请再说一次」，回到步骤 1 |
| **例外流程 E2** | 语音辨识结果无金额 → 提示「未侦测到金额，请手动输入」，自动切换为文字模式 |

**UC3：检视预算比例对比**

| 流程 | 步骤 |
|------|------|
| **主流程** | 1. 使用者切换到 Tab2「总览」<br>2. 系统预设显示「本月」资料<br>3. 系统计算各分类实际花费 ÷ 总花费 = 实际比例<br>4. 并排显示预算比例 vs 实际比例，标注差异<br>5. 超出预算的分类以红色 + ⚠ 标注 |
| **替代流程 A** | 2a. 使用者点选周期下拉，切换为「近 3 个月」→ 系统重新计算并更新图表 |
| **例外流程 E1** | 该周期无任何记账纪录 → 显示「暂无纪录，开始记第一笔吧！」 |
| **例外流程 E2** | 该周期内某分类花费为 0 → 该分类实际比例显示 0%，差异显示「-100%」 |

**UC4：设定月预算与比例**

| 流程 | 步骤 |
|------|------|
| **主流程** | 1. 使用者切换到 Tab3「设定」<br>2. 在「月预算总额」输入 NT$30,000<br>3. 系统自动计算 7 大类的金额上限<br>4. 使用者可拖动滑杆微调各分类比例<br>5. 系统即时校验 7 类比例总和 = 100%<br>6. 点「储存」完成设定 |
| **例外流程 E1** | 4a. 滑杆调整后总和 ≠ 100% →「储存」按钮反灰，提示「比例总和须为 100%」 |
| **例外流程 E2** | 2a. 月预算输入 0 或空白 → 提示「请输入有效的月预算金额」 |

## 4. 资料模型（概要）

```
Transaction（记账纪录）
├── id: UUID
├── amount: Decimal
├── category: Enum (7 classes)
├── note: String?（可空）
├── input_method: Enum (text | voice)
├── created_at: DateTime
└── updated_at: DateTime

BudgetConfig（预算设定）
├── monthly_total: Decimal
├── category_ratios: Map<Category, Percentage>
└── updated_at: DateTime
```

## 5. 模组化架构建议（概要）

```
App
├── Core
│   ├── Storage（本地 SQLite / CoreData）
│   └── Speech（语音辨识 — iOS SFSpeechRecognizer）
├── Features
│   ├── RecordTransaction（记账模组）
│   ├── BudgetOverview（总览/对比模组）
│   └── Settings（设定模组）
└── Shared
    ├── UI Components（共用元件）
    └── Utilities（金额解析、比例计算、周期计算）
```

## 6. 已确认决策

| # | 问题 | 最终决策 |
|---|------|----------|
| Q1 | 语音辨识语种 | MVP 使用繁体中文 + 数字 |
| Q2 | 月预算独立设定 | 每月可独立设定，预设沿用上月数值 |
| Q3 | 跨月结转 | MVP 不做结转，每月独立计算 |
| Q4 | 资料储存 | 纯本地储存（CoreData / SQLite），不加云端 |
| Q5 | 语音辨识方案 | iOS 内建 SFSpeechRecognizer |
| Q6 | 周期定义 | 全部使用自然月（本月 / 近3月 / 近6月 / 近12月） |
| Q7 | 记账时间精度 | 记录到分钟 |
